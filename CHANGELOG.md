## [Unreleased]

### Added
- **Monorepo** — restructured into `packages/{api,cli,core,web}` using Bun workspaces; each package scaffolded with `bun init`
- **Agent pipeline** (`packages/core`) — full Extract → Clarify → Resolve → Draft → Generate → ZIP pipeline with SQLite-persisted session state
- **API server** (`packages/api`) — Hono HTTP server; `/health` + `/sessions/*` routes
- **CLI** (`packages/cli`) — Ink (React for CLIs) terminal UI; state machine drives the pipeline interactively; compiles to standalone binary
- **Web scaffold** (`packages/web`) — Bun fullstack server (`import from "./index.html"`); React + shadcn/ui; proxies `/api/*` to the API server
- **Multi-provider LLM** — Vercel AI SDK factory in `packages/core/src/lib/llm.ts`; supports Anthropic (default), OpenAI, Google, Mistral, Cohere, AWS Bedrock, Azure, and any OpenAI-compatible endpoint (Ollama) via `LLM_PROVIDER` env var
- **bun-native DB migrate** — `drizzle-kit generate` + `drizzle-orm/bun-sqlite/migrator`; avoids `better-sqlite3` incompatibility with Bun
- **Per-package tsconfig** — each package has its own `tsconfig.json` generated by `bun init`; root tsconfig removed
- **Biome 2** — Tailwind CSS support (`css.parser.tailwindDirectives`); overrides for web template files; `bun run check:fix` auto-fixes
- **`bun --filter` scripts** — all root scripts delegate to packages via `bun --filter @groundzero/<pkg> run <script>`; no direct path references

### Changed
- Database artifacts relocated to `packages/core/` — schema, migrations, `drizzle.config.ts`, and `.db` files all co-located
- DB file path resolved via `import.meta.url` (not CWD-relative) so it works regardless of invocation directory
- All `process.env` → `Bun.env` across server packages

### Removed
- Flat `src/` layout — all code now lives in packages
- Root `tsconfig.json` — no longer needed
- `@clack/prompts` — replaced by Ink
